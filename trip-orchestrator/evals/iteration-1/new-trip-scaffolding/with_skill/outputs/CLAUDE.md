## Agent Workflow Notes

### Session Protocol

**At session start:**
1. Read CLAUDE.md (this file) — current phase, status, open questions
2. Read TASKS.md — what's active, next, blocked
3. Read the latest session log (linked above)
4. Summarize briefly (10-15 lines): current phase, active tasks, suggested focus
5. Ask user if they want to go with that or redirect

**At session end:**
1. Write a session log to `memory/sessions/session-log-{date}-{letter}.md`
2. Update CLAUDE.md with new phase (if changed), last session pointer, open questions
3. Reconcile TASKS.md: move completed tasks to Done section with strikethrough + date

### Session Log Format

Each session log includes:
- **Summary:** 2-3 sentences
- **Changes Made:** Bullet list of files created/modified
- **Key Decisions:** What was decided and why
- **Open Items:** Anything unresolved
- **Next Session:** What to focus on next

### General Principles

- **Check TASKS.md first** at session start — it's the task priority list
- **On disk is source of truth** — CLAUDE.md, TASKS.md, memory/ files are the real project state
- **Document decisions in project files** — not in session logs alone
- **Phases are natural progression, not rigid** — they can overlap
- **Research → Guide pipeline:** Research files are broad comparisons. Guides are curated verdicts created *after* user approval. Never create a guide before user input.
